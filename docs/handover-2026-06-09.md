### 2. NUI cache gotcha (important)

The NUI router caches page elements at the JavaScript level (`createRouter()` in `modules/nui_wc2/NUI/nui.js`). `init(element, params, nui)` only runs **once per page per JS context** — subsequent navigations call `element.show?.(params)`. If the first `init()` throws, the cached element stays broken across reloads in the same tab.

**Workaround:** navigate via Projects → click card → Editor → Render & Play to force a fresh init chain. Closing and re-opening the tab also works. The dev-tools "Disable cache" checkbox is the most reliable way to avoid this.

### 1. `<msg>.<split> <label>` header numbering

The user asked: for split messages, show slide headers as `1.0 glm5-chat`, `1.1 glm5-chat`, `1.2 glm5-chat` etc. so it's clear which sub-slides belong to the same source message.

**Design intent:**
- `originalIdx` is the source message index (0-based in the array, displayed as 1-based in the UI: 1, 2, 3, …)
- `splitIdx` is the position within the message (0-based, displayed as 0, 1, 2, …)
- For non-split messages the header just shows `1.0 glm5-chat` (the .0 is always shown so the format is consistent)
- For setup/details/title/end, the header just shows `Setup` / `Details` / `Title` / `End` (no number)
- Applied both in the **player header** and the **slides list row** for consistency

**Why the LLM-removal was right:** the `splitIdx` doesn't need to be stored on the slide. It can be computed at render time by walking the deck once and counting how many previous slides share the same `originalIdx`. This is what I had implemented as a memoized `getSplitIndices()` helper.

**Implementation sketch:**
```javascript
// In web/js/pages/render.js, inside init() or as a top-level function
let splitIndexCache = null;
function getSplitIndices() {
    if (splitIndexCache) return splitIndexCache;
    const map = new Map();
    const seen = new Map(); // originalIdx → running count
    deck.slides.forEach((slide, idx) => {
        if (slide.type !== 'conversation' && slide.type !== 'narration') return;
        if (slide.originalIdx == null) return;
        const count = seen.get(slide.originalIdx) || 0;
        map.set(idx, count);
        seen.set(slide.originalIdx, count + 1);
    });
    splitIndexCache = map;
    return map;
}
```

Then in `loadSlide()`:
```javascript
const splitMap = getSplitIndices();
const splitIdx = splitMap.get(idx) || 0;
let headerLabel;
if (slide.type === 'conversation' && slide.originalIdx != null) {
    const major = slide.originalIdx + 1;
    headerLabel = `${major}.${splitIdx} ${slide.label || slide.speaker}`;
}
```

And in `renderSlideList()` for the row label, swap `${slide.label} #${idx+1}` for `${major}.${splitIdx} ${slide.label}`.

Cache invalidation: clear `splitIndexCache` whenever `deck.slides` is mutated (the `loadProject`, `renderSingleSlide`, and `renderAllSlides` paths).

The `formatTimestamp(iso)` helper returns a two-line-ish "Jun 7, 2026 · 19:37:10" string. Helper definition:
```javascript
function formatTimestamp(iso) {
    if (!iso) return null;
    const d = new Date(iso);
    if (isNaN(d.getTime())) return null;
    const months = ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun', 'Jul', 'Aug', 'Sep', 'Oct', 'Nov', 'Dec'];
    const date = `${months[d.getMonth()]} ${d.getDate()}, ${d.getFullYear()}`;
    const time = `${String(d.getHours()).padStart(2, '0')}:${String(d.getMinutes()).padStart(2, '0')}:${String(d.getSeconds()).padStart(2, '0')}`;
    return `${date} · ${time}`;
}
```

### 3. `*word*` → `word` stripping in TTS / word spans

The user pointed out that audio of `*feel*` is rendered as "asterisk feel asterisk" by nSpeech. The fix: strip `*` markers from the **spoken text** only (the on-screen text keeps them verbatim).

**Implementation sketch:**
```javascript
// Top-level helper in render.js
function stripEmphasisForSpeech(s) {
    if (!s) return s;
    return s.toString().replace(/\*+/g, '');
}

// In getSpokenText():
function getSpokenText(slide) {
    let text = (slide.type === 'title' || slide.type === 'end')
        ? (slide.narration || slide.text || '')
        : (slide.text || slide.narration || '');
    return stripEmphasisForSpeech(text);
}

// In buildWordSpans():
function buildWordSpans(text, tts) {
    const timedSegments = getTimedSegments(tts);
    if (timedSegments.length > 0) {
        return timedSegments.map(segment => {
            const words = segment.words.map(w => {
                const clean = stripEmphasisForSpeech(w.word);
                return `<span class="word future" data-start="${w.startMs}" data-end="${w.endMs}">${escapeHtml(clean)}</span> `;
            }).join('');
            return `<span class="segment future" data-start="${segment.startMs}" data-end="${segment.endMs}">${words}</span> `;
        }).join('');
    }
    if (!text) return '';
    const textWords = stripEmphasisForSpeech(text).split(/\s+/).filter(w => w.length > 0);
    return textWords.map((w, i) => `<span class="word future">${escapeHtml(w)}</span> `).join('');
}
```

## NUI gotcha to remember

The router page cache can hold a broken page element across reloads. Recovery: navigate to a different page first, or close and re-open the tab. The dev-tools "Disable cache" checkbox prevents this entirely during dev.

The 6ad879d no-store middleware helps with **asset** staleness (forces re-fetch of our JS/CSS/HTML), but it can't evict a broken NUI page element that's already in the router's JS-level cache. For that, you still need the tab-reload dance.
